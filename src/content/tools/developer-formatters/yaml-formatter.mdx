## YAML vs JSON
**YAML** is designed to be human-readable and is widely used for configuration files (Docker, Kubernetes, Home Assistant).
**JSON** is stricter and better for machine-to-machine data exchange (APIs, Database storage).

### Key Differences
- **Comments**: YAML supports comments (`#`), JSON does not.
- **Syntax**: YAML uses indentation (whitespace), JSON uses braces `{}`.
- **Strictness**: YAML is more lenient with quotes, while JSON requires double quotes for all keys.
